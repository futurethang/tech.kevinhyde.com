# Dice Baseball V2: UI Specifications

## Overview

Two primary experiences with distinct purposes:

| Experience | Purpose | Connection | Key Actions |
|------------|---------|------------|-------------|
| **Team Builder** | Browse, research, build dream teams | REST API only | Search, compare, save |
| **Live Game** | Competitive multiplayer gameplay | WebSocket | Roll dice, watch plays |

Design Philosophy:
- **Mobile-first** - Primary use case is phone
- **Touch-friendly** - Large tap targets, swipe gestures
- **Fast feedback** - Instant visual response, optimistic UI
- **Baseball aesthetic** - Green/brown palette, stadium feel

## Animation Specifications

### Dice Roll Animation

```
Timeline: 1500ms total

0ms-1200ms: Dice Tumbling
- Two 3D dice bounce and rotate
- Random faces shown rapidly (every 100ms)
- Slight blur effect for motion

1200ms-1500ms: Dice Settle
- Dice slow down, land flat
- Final values shown clearly
- Subtle bounce on landing

1500ms-2000ms: Result Display
- Dice fade slightly
- Outcome text appears below
- Color glow based on outcome
```

### Runner Advancement

```
Duration: 500ms per base

- Dot moves along base path
- Curved arc (not straight line)
- Slight overshoot and settle
- Scoring runner: Flash + fade out at home
```

### Inning Transition

```
Duration: 1500ms

0-500ms: Current half dims
500-1000ms: "MID/END INNING" banner slides in
1000-1500ms: New half UI updates, banner fades
```
