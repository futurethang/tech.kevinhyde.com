Paper grain overlay: SVG feTurbulence generates procedural noise applied as a fixed overlay with mix-blend-mode: overlay at 3.5% opacity. Gives every surface a subtle fibrous texture without any image assets.
Secondary stipple: Fine CSS radial-gradient dots at 1.5% opacity add a second layer of card-stock grain. Two offset dot grids avoid visible patterns.
Ink bleed on text: Warm brown text-shadow (0.3-0.5px spread) on display text simulates ink soaking into paper fibers. Two intensities: normal for headings, heavy for celebration text.
Gloss reflection: Diagonal linear-gradient pseudo-element on cards using mix-blend-mode: overlay. Shifts position on hover for the "tilting a card under light" effect. Foil variant adds a gold tint for special cards.
Edge wear: Inset box-shadow creates a subtle vignette on card frames, mimicking the darker edges of handled cards.
Ink displacement: SVG feDisplacementMap fed by feTurbulence creates micro-distortion on card edges. Applied at very low scale (1.2px) so it reads as "not quite machine-perfect" printing.

Implementation Reference â€” For Claude Code
All texture effects are CSS + inline SVG. No image assets needed. The SVG filters go in a hidden <svg> at the top of the document (or in the React app's root). CSS classes are composable â€” add them to any element.
1. SVG Filter Definitions (add once, at app root)
/* Hidden SVG â€” place in App.tsx or index.html */ <svg width="0" height="0" style="position: absolute"> <defs> <filter id="paper-noise"> <feTurbulence type="fractalNoise" baseFrequency="0.65" numOctaves="4" stitchTiles="stitch" /> <feColorMatrix type="saturate" values="0" /> </filter> <filter id="ink-bleed"> <feTurbulence type="fractalNoise" baseFrequency="0.04" numOctaves="3" seed="2" result="noise" /> <feDisplacementMap in="SourceGraphic" in2="noise" scale="1.2" /> </filter> </defs> </svg>
2. Paper Grain Overlay (global, add once)
/* Fixed overlay â€” covers entire viewport */ .paper-grain { position: fixed; inset: 0; z-index: 9999; pointer-events: none; opacity: 0.035; /* very subtle */ mix-blend-mode: overlay; filter: url(#paper-noise); background: white; } /* Tailwind: fixed inset-0 z-[9999] pointer-events-none opacity-[0.035] mix-blend-overlay bg-white [filter:url(#paper-noise)] */
3. Ink Bleed Text (composable utility class)
/* Normal intensity â€” headings, labels */ .ink-bleed { text-shadow: 0 0 0.5px rgba(139, 69, 19, 0.3), 0.3px 0.3px 0.8px rgba(80, 40, 10, 0.15); } /* Heavy â€” celebration text, banners */ .ink-bleed-heavy { text-shadow: 0 0 1px rgba(139, 69, 19, 0.4), 0.5px 0.5px 1.2px rgba(80, 40, 10, 0.2), -0.2px 0.2px 0.6px rgba(100, 50, 15, 0.1); }

6. Performance Notes
/* PERF: SVG feTurbulence is computed once per filter instance. The paper-grain overlay is a single element â€” negligible cost. The gloss ::after pseudo-element uses GPU-composited properties (background-position), so the hover animation won't trigger layout or paint â€” just composite. Ink bleed text-shadow is sub-pixel and very cheap. The feDisplacementMap filter IS more expensive â€” use it only on card edges or elements that don't animate. Never apply it to elements that reflow frequently. Total texture overhead: ~0.5ms per frame on mid-range mobile. All effects use pointer-events: none so they don't interfere with touch/click handling. */
Design Principles â€” v4 Additions

Texture is Seasoning

Every texture effect is at LOW opacity (1.5-6%). The goal is a feeling, not a visible overlay. If a user notices the paper grain consciously, it's too strong. It should register as "this feels like a card" without them knowing why.

Composable, Not Baked In

Each texture is a separate CSS class (.ink-bleed, .gloss-surface, .aged-edge). Apply them individually or stack them. This keeps the component API clean and lets different card types have different texture profiles.

Gloss = Interactivity Cue

The gloss reflection that moves on hover is also a subtle interactivity signal â€” "this element responds to you." Foil variant on special cards creates a premium feel that rewards collecting good players.

No Image Assets

Every texture is procedural CSS or inline SVG. No .png grain overlays, no texture sprite sheets. This keeps the bundle tiny and means textures scale perfectly to any resolution including Retina.
